# Release Layout

This folder contains the release-first FastLoop distribution scaffold used for GitHub Releases.

- `out/`: versioned GitHub Release artifacts generated by `npm run release:build`
- `installer/`: Inno Setup scripts for the primary Windows setup wizard
- `templates/`: Windows install helpers and future release templates
- `CHECKLIST.md`: release validation checklist before publishing a build
- `SIGNING.md`: current unsigned status and future signing requirements
- `signing.env.example`: signing/release secret placeholders

Current packaging output includes:

- `FastLoop-Windows-x64-Setup.exe` as the primary Inno Setup wizard asset
- `FastLoop-Windows-x64.zip` as the secondary portable asset
- packaged Windows engine runtime
- checksums, release notes, manifest, and install documents
- `release-notes-template.md` as the source scaffold for generated prerelease notes
- `TROUBLESHOOTING.md` plus `Test-FastLoop-HostReadiness.ps1` for “installed but not visible” CEP diagnostics

- setup-wrapper diagnostics under `%LOCALAPPDATA%\FastLoop\Logs`, including `setup-latest.log`, `setup-latest.json`, and helper stdout/stderr captures
- a recovery copy of `FastLoop-Windows-x64.zip` under `%LOCALAPPDATA%\FastLoop\Recovery` when setup reaches the payload stage

Recommended release flow:

```bash
npm run release:build
npm run release:validate
```

GitHub Releases is now the intended end-user entry point. Signing remains the next production-release step.
